## 📖 Overview

The Job Tracker App is a responsive and user-friendly web application designed to help individuals manage their job application process efficiently. It allows users to add, view, update, and delete job application entries, providing a centralized place to keep track of their career pursuits. All data is stored locally in the browser's `localStorage`, ensuring privacy and quick access without needing a backend.

## ✨ Features

-   🎯 **Add New Job Applications**: Easily input details like company name, job title, status (e.g., applied, interview, rejected), and application date.
-   📋 **View All Applications**: A clean, sortable, and filterable list displays all tracked job applications.
-   ✏️ **Update Application Status**: Modify the status of an application as it progresses through the hiring pipeline.
-   🗑️ **Delete Applications**: Remove old or irrelevant job entries with a single click.
-   🔍 **Filter & Sort**: Organize job applications by status, company, or date to quickly find specific entries.
-   💾 **Local Data Persistence**: All job data is securely stored in your browser's `localStorage`, ensuring your entries are saved between sessions.
-   📱 **Responsive Design**: Optimized for various screen sizes, providing a seamless experience on both desktop and mobile devices, thanks to Tailwind CSS and Daisy UI.

## 🚀 Quick Start

This project is a static web application and does not require any build tools or server-side setup.

### Prerequisites
-   A modern web browser (e.g., Chrome, Firefox, Edge, Safari).

### Installation

1.  **Clone the repository**
    ```bash
    git clone https://github.com/satyajtus14/Milestone-04-Job-Tracker-Apps.git
    cd Milestone-04-Job-Tracker-Apps
    ```

2.  **Open in your browser**
    Simply open the `index.html` file in your preferred web browser.
    ```bash
    # Example for macOS
    open index.html
    # Example for Windows
    start index.html
    ```
    Alternatively, you can drag and drop `index.html` into your browser.

## ⚙️ Configuration

This application is primarily client-side and uses `localStorage` for data persistence. There are no server-side environment variables or complex configuration files required for basic operation.

### Tailwind CSS Configuration
-   `tailwind.config.js`: This file is intended for customizing Tailwind CSS. In this project, it exists but appears to be empty, meaning the default Tailwind and Daisy UI configurations are likely used, or the `tailwind.css` file is pre-compiled.

## 🔧 Development

Development involves directly editing the HTML, JavaScript, and CSS files.

### Styling with Tailwind CSS & Daisy UI
To make changes to the styling using Tailwind CSS and Daisy UI, you would typically need to:
1.  Install Node.js and npm/yarn.
2.  Install Tailwind CSS and Daisy UI as dev dependencies.
3.  Configure `tailwind.config.js`.
4.  Compile your CSS using Tailwind CLI or a build tool.

*Note: The current setup includes a pre-compiled `tailwind.css` file. If you intend to modify Tailwind/Daisy UI styles, you may need to set up a local build environment for Tailwind CSS.*
